# Triumphoby OSINT

`Triumphoby OSINT` — это desktop-приложение на Python с графическим интерфейсом для базовых OSINT-задач.
Проект объединяет несколько практичных модулей в одном окне:

- dorking по поисковым системам;
- поиск по username на популярных платформах;
- базовая проверка телефонного номера с расширением через API-источники.

Приложение ориентировано на быстрый ручной ресерч: ввел цель, получил структурированные результаты, открыл нужные ссылки и продолжил проверку.

## Для чего этот проект

`Triumphoby OSINT` помогает ускорить первичную проверку:

- никнеймов (где может быть занят профиль);
- телефонов (формат, оператор, регион, дополнительные подсказки);
- поисковых следов по dork-шаблонам.

Подходит для исследовательской работы, расследований в рамках закона, подготовки аналитики и повседневного OSINT-workflow.

## Ключевые возможности

### 1) Dorking

- выбор поисковой системы (`Google`, `DuckDuckGo`, `Bing`, `Yandex`, `Baidu`);
- готовые dork-шаблоны + ручной таргет;
- запуск одного запроса или перебор всех шаблонов;
- прокси, таймаут, User-Agent, лимит результатов;
- вывод результатов прямо в интерфейсе.

### 2) Name Search

- проверка username по списку популярных сервисов;
- визуальная индикация найдено/не найдено;
- быстрый переход по найденным URL;
- прогресс выполнения в реальном времени.

### 3) Phone Check

- нормализация и валидация номера (`phonenumbers`);
- базовые данные: международный формат, регион, оператор, таймзоны, тип номера;
- HLR lookup (при наличии API-ключей);
- быстрые ссылки для ручной проверки в соцсетях и Google;
- блок для будущих интеграций (Truecaller/GetContact/Eyecon и др.).

### 4) Settings

- прозрачность и blur-режим интерфейса;
- сохранение параметров в `settings.json`;
- хранение API-ключей для модуля Phone Check.

## Технологический стек

- Python 3.10+
- `customtkinter` — интерфейс
- `requests` + `beautifulsoup4` + `lxml` — HTTP и парсинг
- `aiohttp` — асинхронные сетевые запросы
- `phonenumbers` — проверка и разбор номера
- `BlurWindow` — эффект blur окна (Windows)

## Как это устроено

- `main.py` — точка входа, окно приложения, навигация между страницами.
- `ui/pages.py` — логика экранов (`Dorking`, `Name Search`, `Phone Check`, `Settings`).
- `ui/widgets.py` — переиспользуемые виджеты (кнопки, dropdown, кнопки sidebar).
- `core/search.py` — движок dork-поиска и парсинг выдачи поисковиков.
- `core/username_checker.py` — проверка username по набору сервисов.
- `core/phone_checker.py` — проверка номера и интеграции внешних источников.
- `config.py` — палитра, dork-шаблоны, поисковики, настройки по умолчанию.

## Установка

### 1) Клонируйте репозиторий

```bash
git clone <https://github.com/Solduramigopmo/Triumphoby-OSINT>
cd triumphoby
```

### 2) Создайте и активируйте виртуальное окружение

Windows (PowerShell):

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Linux/macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3) Установите зависимости

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Настройка `settings.json`

Файл создается/обновляется приложением автоматически. Ключевые параметры:

- `bg_opacity` — прозрачность окна;
- `blur_type` — `None | Standard | Acrylic`;
- `proxy_enabled`, `proxy_http`, `proxy_https` — прокси для dorking;
- `timeout`, `max_results`, `delay_between_requests` — сетевые лимиты;
- `user_agent` — User-Agent заголовок;
- `smsc_login`, `smsc_password` — доступ к SMSC.ru;
- `numverify_api_key` — ключ Numverify;
- `telegram_api_id`, `telegram_api_hash` — задел под Telegram-интеграции.

- базовый UI и навигация между модулями;

- blur-эффекты завязаны на Windows-окружение.

## Известные нюансы

- API-ключи сохраняются локально в `settings.json`; если публикуете проект, не коммитьте приватные ключи.
- Поисковые движки могут временно отдавать CAPTCHA/блок-страницы.
- Некоторые страницы и сервисы меняют HTML, из-за чего парсеры требуют периодической донастройки.

## Частые проблемы

### Приложение не запускается из-за `BlurWindow`

- это опциональная зависимость для визуального эффекта;
- можно установить пакет повторно или выбрать `blur_type = None` в настройках.

### `Name Search` показывает мало попаданий

- это нормально для редких username;
- часть платформ может требовать авторизацию или отдавать нестандартные ответы.

### `Phone Check` возвращает "No API keys configured"

- добавьте `SMSC.ru` или `Numverify` ключи в `Settings`;
- без ключей модуль показывает только базовую информацию и ссылки для ручной проверки.
